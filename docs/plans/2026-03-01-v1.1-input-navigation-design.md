# v1.1 — Input & Navigation Design

> Make the first 10 seconds feel seamless.

## Scope

Six features, shipped in three pairs ordered by user impact:

1. **Lenient parser + Better parse errors** — fix the input pipeline
2. **Breadcrumbs + Expand/collapse buttons** — improve tree navigation
3. **Format/minify toggle + Keyboard shortcuts legend** — polish the output

---

## Pair 1: Lenient Parser + Better Parse Errors

### Lenient Parser

**Goal:** Accept malformed JSON that developers commonly encounter — trailing commas, single quotes, unquoted keys, Python booleans (`True`/`False`/`None`), and comments (`//`, `/* */`).

**Approach:** Add `jsonrepair` as a dependency. The parsing pipeline becomes:

1. Try `JSON.parse(input)` — fast path for valid JSON
2. On failure, run `jsonrepair(input)` to produce valid JSON
3. Parse the repaired string with `JSON.parse`

**State changes:**

```typescript
interface RepairInfo {
  wasRepaired: boolean
  originalInput: string
  repairedInput: string
}
```

`jsonStore` gains an optional `repairInfo` field alongside `parseResult`.

**Worker path:** Same fallback — `JSON.parse` first, `jsonrepair` on failure. The `jsonrepair` dependency loads in the worker bundle.

**UX:** Subtle wrench icon in the tree toolbar. Hover/click reveals a tooltip listing what was fixed (derived by diffing original vs repaired). Does not interrupt the flow.

### Better Parse Errors

**Goal:** When JSON is truly unparseable (both `JSON.parse` and `jsonrepair` fail), show the error in context with line/column information.

**State changes:**

```typescript
interface ParseErrorInfo {
  message: string
  line?: number
  column?: number
  context?: string
}
```

`jsonStore.parseError` changes from `string` to `ParseErrorInfo | null`.

**Line/column extraction:** Parse the position from the native `SyntaxError` message. Browsers include position info in varying formats; extract with regex patterns for Chrome, Firefox, and Safari.

**UX:** Red-tinted card in the main content area showing the error message, the problematic line with a caret pointing to the error column, and 2-3 surrounding lines for context.

## Pair 2: Breadcrumbs + Expand/Collapse Buttons

### Breadcrumb Path Bar

**Goal:** Show a clickable trail from root to the selected node: `$ > data > users > [0] > name`.

**Architecture:**
- Derives from `selectedNodeId` — walk up via `parentId` to build path segments
- Clicking a segment calls `selectNode` + `expandToNode` to select and scroll to that node
- Only visible when `selectedNodeId` is not null

**New component:** `Breadcrumbs` — rendered inside a new `TreeToolbar` above `TreeView`.

**Overflow:** Horizontal scroll with rightmost (deepest) segment always visible. Subtle fade on left edge for overflow indication.

**Styling:** Mono-spaced text, muted chevron separators, segments highlight on hover.

### Expand/Collapse All Buttons

**Goal:** Visible UI buttons for expand/collapse all (keyboard shortcuts already work but aren't discoverable).

**Architecture:**
- Two icon buttons: `ChevronsDown` (expand all), `ChevronsUp` (collapse all) from lucide-react
- Invoke `expandAll()` / `collapseAll()` from `jsonStore`
- Right-aligned in the `TreeToolbar`, same row as breadcrumbs

**Layout:**

```
┌──────────────────────────────────────────────────┐
│ [breadcrumbs...                    ] [↕↑] [↕↓]  │
├──────────────────────────────────────────────────┤
│ tree content (virtualized)                       │
```

`TreeToolbar` renders only when `parseResult` exists in explore mode.

## Pair 3: Format/Minify Toggle + Keyboard Shortcuts Legend

### Format/Minify Toggle

**Goal:** Switch the raw value display in the detail panel between pretty-printed and compact.

**State:** New `rawViewFormat: 'pretty' | 'minified'` field in `uiStore`.

**Architecture:**
- Toggle button in `Toolbar` (next to theme toggle and share)
- `DetailPanel` reads `rawViewFormat` to decide between `JSON.stringify(value, null, 2)` and `JSON.stringify(value)`
- Icons: `Maximize2` (pretty) / `Minimize2` (minify) from lucide-react

**Scope:** Affects only the Value section of `DetailPanel`. Tree view is unchanged.

### Keyboard Shortcuts Legend

**Goal:** Discoverable, context-sensitive shortcut reference.

**Architecture:**
- `?` button in `Toolbar` (right side, after theme toggle)
- Opens `ShortcutsModal` — centered modal with semi-transparent backdrop
- Reads `mode` from `uiStore` to show relevant shortcuts

**Sections:**

**Global** (all modes):
- `Ctrl+K` → Focus search
- `?` → Show shortcuts

**Explore mode:**
- `Ctrl+E` → Expand all
- `Ctrl+Shift+E` → Collapse all

**Diff mode** and **Query mode** sections populate as shortcuts are added.

**Interaction:** Opens with `?` button or `Shift+?` key. Closes with `Esc`, clicking outside, or pressing `?` again.

**Styling:** Dark card with monospaced `kbd` pill elements, grid layout of shortcut → description pairs.
